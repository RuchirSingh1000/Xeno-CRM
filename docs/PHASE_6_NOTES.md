# Phase 6 — Cross-Campaign Analytics

**Goal:** the portfolio view. AI insight per campaign already landed in Phase 5; Phase 6 is what a CMO/Growth lead opens every Monday to see what's working across the whole engagement program.

## What exists now

### Backend
- `app/services/analytics.py` — aggregation functions over the same source tables we already have (no warehouse, no ETL). Six top-level views:
  - `overview()` — hero KPIs: campaigns, customers reached, total revenue, delivery/CTR/conversion rates
  - `channels()` — per-channel performance with revenue and revenue-per-send
  - `campaigns_leaderboard()` — sortable per-campaign table, with `is_ai_planned` flag so AI vs hand-built comparison is visible
  - `failures()` — failure-reason breakdown + per-channel mix + webhook integrity counters
  - `ai_usage()` — by purpose: runs, fallback rate, avg latency. By provider for tracking the demo's actual model spend
  - `revenue_timeline()` — per-campaign revenue contribution

### API routes
- `/analytics/overview`, `/analytics/channels`, `/analytics/campaigns-leaderboard`, `/analytics/failures`, `/analytics/ai-usage`, `/analytics/revenue-timeline` — each independently queryable
- `/analytics/dashboard` — one call that returns everything (used by the UI in a single round-trip)

### Frontend
- **`/analytics`** page with five blocks:
  1. **Hero KPI strip** — total revenue (₹), campaigns run, customers reached, conversion rate, failure rate
  2. **Portfolio funnel** (left) — sent → delivered → clicked → converted → failed, with stage percentages
  3. **Channel performance table** (right) — sent / delivery / CTR / conversion / revenue / ₹-per-send per channel
  4. **Campaign leaderboard** — sortable by revenue, conversion, CTR, delivery, or targeted. AI-planned campaigns get an "AI" badge.
  5. **Failure analysis** — bar chart by reason + webhook integrity panel (processed / duplicates / invalid sigs / no-comm)
  6. **AI layer health** — total runs, fallback rate, providers used, per-purpose breakdown with ok/retry/fallback split bar
  7. **Revenue by campaign** — sorted bar chart

## Defense one-liners

- **"How does this scale to 10M customers?"** Today every aggregate is a query-time view over the operational tables. At scale you'd precompute rolling aggregates into a `campaign_stats` row that updates on event ingest (the webhook receiver is the right hook). The dashboard endpoint becomes a single SELECT. Phase 7 documents the migration path.
- **"Why no warehouse?"** Premature. Snowflake/BigQuery makes sense at >1B events. For a Brewhouse-sized brand the operational DB is plenty fast and the analytics stay correct under live operations (no ETL lag).
- **"Revenue attribution?"** Conversion value comes from the channel simulator's webhook payload via `metadata.conversion_value_inr`, summed at query time with SQLite `json_extract`. In production this would be a real `Order.attributed_to_campaign_id` join, which we already have the schema room for.
- **"What's the AI badge on campaigns telling me?"** It's tagging campaigns whose Segment was generated by AI OR whose plan came through `/campaigns/ai-plan`. Lets a Growth lead see — empirically — whether AI plans actually convert better than hand-built ones. That's the kind of question a CMO actually asks.
- **"Webhook integrity counters"** — Duplicates ignored (idempotency working), invalid signatures (HMAC rejection), no-communication-for-event (race or stale payload). These are operator metrics for the FDE on call, surfaced alongside the marketer metrics because Phase 6 serves both audiences.

## What's deliberately NOT in Phase 6

- **No charts library.** Every visual is hand-rolled with Tailwind divs. Saves bundle size, matches the design system, and is one less dependency for the interviewer to ask about.
- **No date filtering.** The seed data spans 540 days but the analytics view is "all-time" by default. Adding a date picker is Phase 7 polish.
- **No segment performance.** A "by segment" leaderboard (which segments convert best) is the obvious next view but requires linking campaigns back to segments deeply. Phase 7 if time permits.
- **No cohort retention.** Beyond scope for a one-week build.
